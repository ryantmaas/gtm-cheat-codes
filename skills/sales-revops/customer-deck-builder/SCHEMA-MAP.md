# Customer Deck Builder Schema Map

Use this map to connect the skill to approved systems without exposing private implementation details.

## Account Context

- Company name
- Domain
- Account ID
- Plan or package
- Renewal date or contract timing
- Account owner or customer-success owner
- Segment, region, or industry

## Usage Context

- Recent task or automation volume
- Active workflows
- Active users or builders
- Top apps and app categories
- Feature adoption
- Usage trend
- Error, pause, or optimization signals when approved

## Meeting Context

- Call type
- Meeting goal
- Attendee roles
- Open questions
- Customer priorities
- Prior meeting notes

## Deck Guidance

- Approved call-type template
- Approved use-case library
- Approved customer proof
- Brand or presentation guidance
- Gamma generation settings

## Do Not Store In This Public Package

- Raw SQL
- Private table names
- Customer exports
- Personal emails
- Private notes
- Internal access instructions
- Customer-specific decks
